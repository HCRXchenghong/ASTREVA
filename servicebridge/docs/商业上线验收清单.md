# 商业上线验收清单

本文档用于正式交付前复核，避免“本地能跑”和“生产可用”之间出现遗漏。

## 1. 代码质量门禁

上线前必须通过：

```bash
make commercial-acceptance
```

该命令会执行：

- Go 后端全量测试。
- 用户端 Web 文件存在性检查。
- 开发与生产 Docker Compose 配置检查。
- 生产环境变量预检。
- 后端生产镜像构建。
- 本地后端启动、上传、指标和 WebSocket 烟测。

如果本机暂时不能构建 Docker 镜像，可先执行：

```bash
SKIP_DOCKER_BUILD=true make commercial-acceptance
```

## 2. 生产资源确认

上线前必须由项目方提供：

- 用户端域名，例如 `service.your-domain.com`。
- API / WebSocket 域名，例如 `api.your-domain.com`。
- 飞书事件回调公网地址，例如 `https://api.your-domain.com/api/integrations/feishu/events`。
- TLS 证书。
- PostgreSQL 生产实例和备份策略。
- Redis 生产实例。
- `DATA_ENCRYPTION_KEY`，至少 32 字符随机值，上线后不要随意更换。
- 管理员和客服初始强密码。
- 飞书自建应用 `App ID`、`App Secret`、`Verification Token`、`Encrypt Key` 和默认客服群 `chat_id`。
- OpenAI 或 OpenAI 兼容接口 Key。
- 对象存储或共享卷方案；生产推荐 S3 / MinIO 兼容对象存储加 CDN。
- 监控采集系统，用于拉取 `/metrics`。

## 3. 生产配置底线

生产环境必须满足：

- `STORE_DRIVER=postgres`
- `DATABASE_URL` 指向生产 PostgreSQL。
- `REDIS_ADDR` 指向生产 Redis。
- `CORS_ALLOWED_ORIGINS` 只允许真实用户端域名。
- `TRUSTED_PROXY_CIDRS` 只允许真实负载均衡或 Nginx 网段。
- `RATE_LIMIT_ENABLED=true`
- `SECURITY_HEADERS=true`
- `METRICS_BEARER_TOKEN` 必须设置。
- 飞书核心配置必须能通过后台接口保存，并通过测试消息验证。
- 默认密码、示例域名、`change-me` 占位值不允许进入生产。

## 4. 容量验收

正式承诺 5000 WebSocket 并发前，在目标服务器执行：

```bash
HTTP_BASE=https://api.your-domain.com \
WS_BASE=wss://api.your-domain.com \
DURATION=60s \
STEPS="100 500 1000 2500 5000" \
scripts/loadtest/acceptance-5000.sh
```

验收标准：

- 每个阶梯 `connected` 必须等于目标连接数。
- `failed=0`。
- 滚动重启单个后端实例后，客户端能重连并通过历史接口补齐消息。
- PostgreSQL 和 Redis 指标没有持续性错误或明显资源打满。

## 5. 发布确认

用户端 Web：

- 生产页面不依赖外部 CDN。
- `customer-service-api-base` 或同源反代指向生产 API。
- 手机浏览器和电脑浏览器都能创建会话、发文字、发图片、评价。

飞书直连：

- 飞书 URL 校验通过。
- 访客发送消息后，默认飞书群收到消息。
- 客服直接回复飞书消息后，网站访客实时收到回复。
- 同一飞书事件重复投递不会重复生成客服消息。
- 后端重启后，飞书消息绑定仍能从 PostgreSQL 恢复。

## 6. 交付边界确认

第一阶段不包含语音、视频、工单、CRM、多渠道聚合、多租户计费和端到端加密。若要进入这些能力，需要先更新 `客服系统边界说明.md`，再单独排期。
