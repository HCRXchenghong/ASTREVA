# 飞书直连客服

本文档记录飞书内直接回复版客服链路。

## 接入模式

飞书是人工客服工作台。网站访客发送消息后，后端把消息推送到默认飞书群；客服在飞书里回复该消息，或在任意消息中带 `#conv_xxx`，后端会把飞书消息落库为客服回复并实时推送给网站访客。

## 飞书开放平台配置

需要在飞书开放平台创建自建应用：

- 开启机器人能力。
- 开通发送消息接口权限。
- 订阅接收消息事件 `im.message.receive_v1`。
- 将事件订阅请求地址配置为：

```text
https://service.example.com/api/integrations/feishu/events
```

- 支持事件订阅加密；如飞书侧启用加密，请同步配置 `FEISHU_ENCRYPT_KEY`。
- 将应用机器人加入默认客服群，并记录群 `chat_id`。

## 后台配置

飞书配置通过后端管理接口保存到数据库，环境变量只作为首次启动兜底。

```http
PATCH /api/admin/integrations/feishu
Authorization: Bearer <admin_token>
Content-Type: application/json

{
  "enabled": true,
  "base_url": "https://open.feishu.cn",
  "app_id": "cli_xxx",
  "app_secret": "xxx",
  "verification_token": "xxx",
  "encrypt_key": "xxx",
  "default_chat_id": "oc_xxx",
  "agent_id": "agent_feishu",
  "timeout_seconds": 8
}
```

测试配置：

```http
POST /api/admin/integrations/feishu/test
Authorization: Bearer <admin_token>
Content-Type: application/json

{
  "text": "飞书客服接入测试"
}
```

## 消息链路

1. 访客在网站聊天窗口发送消息。
2. 后端调用飞书发送消息 API，把访客消息发到默认客服群。
3. 后端保存飞书 `message_id -> conversation_id` 绑定。
4. 客服在飞书中回复该消息。
5. 飞书把消息事件 POST 到 `/api/integrations/feishu/events`。
6. 后端根据 `parent_id/root_id/message_id` 查找会话；找不到时从文本中的 `#conv_xxx` 兜底。
7. 后端使用虚拟客服 `agent_feishu` 接管会话并落库客服消息。
8. 后端通过 WebSocket 把客服消息推送给网站访客。

## 当前限制

- 飞书用户暂不映射为系统客服个人身份，统一归属到 `agent_feishu`。
- 飞书卡片按钮后续再加；当前以文字回复为主。

## 验收

飞书 URL 校验：

```bash
curl -X POST http://localhost:8080/api/integrations/feishu/events \
  -H 'Content-Type: application/json' \
  -d '{"challenge":"test_challenge","token":"<后台保存的 verification_token>","type":"url_verification"}'
```

应返回：

```json
{"challenge":"test_challenge"}
```

端到端验收：

- 网站访客发一条消息，飞书群能收到。
- 在飞书中直接回复该条消息，网站访客能收到回复。
- 重复投递同一飞书事件，不会出现重复客服消息。
